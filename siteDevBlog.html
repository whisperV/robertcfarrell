Site development updates

August 6, 2017 / by Robert C. Farrell

This website is a project I am working on while job hunting. I am hoping it will help me stand out and land a rewarding and fulfilling position at a company interested in the skills demonstrated by this page. This page, specifically, is just for following along with the creation of the site, and will be updated outside of the tax blog.

Disclaimer: Nothing on this site should be used for anything meaningful. This is just a pet project with the only intention being to showcase my skills.

So this site is admittedly, a bit of a mess. The first work was done on July 2, 2017 and was basically just my resume. In about a month the site not has a landing page, resume, Blog space, and tools section. As it stands the site is not fully functional, and is not released to the public or even indexed on the web. It will be considered "presentable" when the one blog post which is up is correct and the IRA contribution calculator is error free. Once those are addressed, the functionality and interactivity of my resume as well as overall design and aesthetic will updated.

The two major components still under construction are the Blog post and the IRA contribution calculator. The Blog post is just a matter of sitting down and proofreading the work one or two more time, then adding any necessary citations. The IRA contribution calculator is a bit more work, but progessing nicely. The functionality is priority, with design and style taking a backseat to accuracy. As of this update, the answers are still only occasionally correct. There are a fair number of factors which go into determining the maximum a person can contribute, and getting them all to come together is proving to be a game of whack a mole or errors. It should be noted, I have extraordinary limited formal training in programming (2 classes, 1 at a community college - although I learned more there) and the onlyt web development training I have is the Udemy web development Bootcamp, which I got on sale for a great price.

Since this entry is being made prior to the website being 'released', it is mostly just for those interested in the development process, and will mostly just have an update if work was done that day, and a breif run down of what was changed, and any 'problems of the day' for any challenges encountered.

The work done today

Today was work on the IRA contribution calculator. An age question was added. If someone is over 50 years old, they get to add $1,000.00 to their contribution limit. That is per taxpayer so married filing joint will add $2,000.00 if both filers are over 50, only $1,000 if one one is over 50. A discalimer was also added in case someone does stumble on this site somehow. At this time, the contribution amounts for some filers will calculate correctly. If you are Single and contributing to a Roth it should tell you the correct amount. MFJ filers should be accurate as well, but needs more extensive testing.

The First Problem of the Day

Today's big problem was with getting the age question to alter the contribution limit correctly. And only one time. There are 3 possible scenarios with 3 possible outcomes. 1. Both user and spouse are over 50 2. Just user or spouse is over 50 3. Neither is over 50. This seems straighforward enough and writing the selection and calculation was not extremely difficult. The problem was with the HTML inputs and what happened when a user clicked an option multiple times. The HTML used is:

                                      
Are you over 50 years old? 
                                        
    
                                            
  •  Yes 
    • 
                                            
  •  No 
    • 
                                            
  •  Check here if your spouse is over 50
    •

This bit has two function calls, which use the input's value to determine how to adjust the general contribution limit. The question about a spouse only shows when Married Filing Jointly is chosen, and for each of these options a $1,000 is added to the general limit. The general limit is a variable which changes depending on filing status, age and about a dozen other things, but starts at $5,500 for each taxpayer. Each person over 50 increases this amount by $1,000. So a married filing joint couple over 50 has a general limit of $13,0000. $5,500 each just because, and an extra $1,000 each for being over 50. The biggest issues were with the function calls, since each click called the function. The solution was to disable Yes or No if thier counterpart was checked. Since the spouse age is a different function, and on a checkbox input, it could act independently of the yes/no option. Here is the jquery code for the over50() function:

Even as an amatuer, I can tell there is too much going on here and there is likely a much easier way to accomplish the same task. If this was the only thing to do, then I would spend the time finding more optimal ways to answer the same question, however since this is a project I am doing while unemployed, it suffices for now. What is does is disallow additional clicking which means the function can't be called again by the same radio selector. This code will bug me for a while, and it hopefull one day I will be in a position to update it to be more efficient. In the meantime it is ugly and working, with the emphasis on working.
